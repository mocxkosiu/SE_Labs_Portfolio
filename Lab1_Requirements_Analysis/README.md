# Lab 1 Conclusions

This lab introduced the fundamentals of Software Requirements Specification (SRS) design and documentation. During the exercise, I learned how to structure an SRS document according to common software engineering practices and how to identify the key elements required for a complete system specification.

## Learning Outcomes

After completing this lab, I am able to:

* Develop and maintain Software Requirements Specification (SRS) documents.
* Distinguish between functional and non-functional requirements.
* Define system scope, purpose, and target user groups.
* Identify external interfaces and system constraints.
* Organize software requirements using a structured and professional format.
* Apply software engineering principles during the requirements analysis phase.
* Create documentation that can serve as a foundation for future software development activities.

## Summary

The knowledge gained during this lab will be useful in future software engineering projects, particularly during the planning, analysis, and documentation stages of the software development lifecycle.
