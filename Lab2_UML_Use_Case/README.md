# Lab 2 Conclusions

This lab introduced the fundamentals of UML Use Case Diagrams and their role in requirements analysis. The exercise involved identifying system actors, defining system functionality through use cases, and modeling relationships between use cases.

## Learning Outcomes

After completing this lab, I am able to:

* Identify primary and secondary actors interacting with a software system.
* Define system functionality using UML use cases.
* Distinguish between mandatory (`<<include>>`) and optional (`<<extend>>`) use case relationships.
* Establish clear system boundaries and model interactions between actors and the system.
* Create UML Use Case Diagrams that accurately represent functional requirements.
* Use Use Case Diagrams to complement and refine Software Requirements Specification (SRS) documentation.
* Apply UML modeling techniques during the requirements analysis phase of software development.

## Summary

The Use Case Diagram created during this lab was incorporated into the project's SRS document. This diagram provides a visual representation of the system's functionality and stakeholder interactions, making the requirements easier to understand and communicate among project participants.
