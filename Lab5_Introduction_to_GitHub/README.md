# Lab 5 Conclusions

This lab introduced the fundamentals of version control systems, command-line tools, Git, and GitHub. The exercises focused on understanding how source code and project files can be tracked, managed, and shared efficiently in both individual and collaborative software development environments.

## Learning Outcomes

After completing this lab, I am able to:

* Understand the purpose and benefits of version control systems.
* Explain the role of version control in software development projects.
* Use basic command-line interface (CLI) commands to navigate and manage files and directories.
* Initialize and manage local Git repositories.
* Track file changes using Git.
* Create commits and maintain project history.
* Work with branches and understand their purpose in parallel development.
* Connect local repositories to remote repositories hosted on GitHub.
* Push and pull changes between local and remote repositories.
* Use GitHub as a platform for source code hosting, collaboration, and project management.

## Topics Covered

### Version Control

* Purpose and advantages of version control systems.
* Tracking changes in project files.
* Maintaining project history and supporting collaboration.

### Basic CLI Commands

* File and folder management.
* Command-line interaction with development tools.

### Git

* Repository creation and initialization.
* Staging and committing changes.
* Viewing repository status and history.
* Branch management.

### GitHub

* Creating and managing remote repositories.
* Synchronizing local and remote repositories.
* Sharing projects and collaborating with other developers.

## Summary

The knowledge gained during this lab provides a foundation for modern software development workflows. Git and GitHub enable efficient version tracking, collaboration, and project management, while command-line skills improve productivity and understanding of development environments. These tools are essential for both individual and team-based software engineering projects.
