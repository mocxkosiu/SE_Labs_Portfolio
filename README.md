# Software Engineering Laboratories – Summary

This repository contains the work completed during six Software Engineering laboratory sessions, covering the fundamental stages of software development and design.

## Labs Overview

* **Lab 1:** Software Requirements Specification (SRS) – requirements analysis and documentation.
* **Lab 2:** UML Use Case Diagrams – modeling actors and system functionality.
* **Lab 3:** UML Class Diagrams – object-oriented design and class relationships.
* **Lab 4:** UML Behavioral Diagrams – Sequence, State, and Activity Diagrams.
* **Lab 5:** Git and GitHub – version control and collaborative development.
* **Lab 6:** Test-Driven Development (TDD) – implementation and testing of a Traffic Light Controller.

## Learning Outcomes

Throughout these labs, I gained practical experience in:

* Requirements engineering and documentation.
* UML modeling and software design.
* Object-oriented analysis and design principles.
* Version control using Git and GitHub.
* Automated testing and Test-Driven Development.
* Applying software engineering methodologies across the software development lifecycle.

## Summary

The laboratories provided a solid foundation in software engineering practices, from requirements gathering and system modeling to implementation, version control, and testing.
